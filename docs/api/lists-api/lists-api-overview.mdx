<div id="lists-api-overview"></div>

Lists can be used with detection rule <DocLink id="serverlessSecurityExceptionsApiOverview">exceptions</DocLink>
to define values that prevent a rule from generating alerts.

Lists are made up of:

* **List containers**: A container for values of the same ((es))
    [data type](((ref))/mapping-types.html). The following data types can be used:

    * `boolean`
    * `byte`
    * `date`
    * `date_nanos`
    * `date_range`
    * `double`
    * `double_range`
    * `float`
    * `float_range`
    * `half_float`
    * `integer`
    * `integer_range`
    * `ip`
    * `ip_range`
    * `keyword`
    * `long`
    * `long_range`
    * `short`
    * `text`

* **List items**: The values used to determine whether the exception prevents an
    alert from being generated.

All list items in the same list container must be of the same data type, and
each item defines a single value. For example, an IP list container, named
`internal-ip-addresses-southport`, contains five items, where each item defines
one internal IP address:

1. `192.168.1.1`
1. `192.168.1.3`
1. `192.168.1.18`
1. `192.168.1.12`
1. `192.168.1.7`

To use these IP addresses as values for defining rule exceptions, use the
<DocLink id="serverlessSecurityExceptionsApiOverview">Exceptions API</DocLink> to create an
<DocLink id="serverlessSecurityExceptionsApiCreateExceptionItem">exception item</DocLink> that references the
`internal-ip-addresses-southport` list.

<DocCallOut title="Note">
Lists cannot be added directly to rules, nor do they define the operators
used to determine when exceptions are applied (`is in list`, `is not in list`).
Use an <DocLink id="serverlessSecurityExceptionsApiCreateExceptionItem">exception item</DocLink> to define the
operator and associate it with an <DocLink id="serverlessSecurityExceptionsApiCreateContainer">exception container</DocLink>.
You can then add the exception container to a rule's `exceptions_list` object.
</DocCallOut>

<DocCallOut title="Important" color="warning">
Before you can create lists, you must create `.lists` and `.items`
indices for the ((kib)) space (see <DocLink id="serverlessSecurityListsIndexApiOverview">Lists index endpoint</DocLink>).
</DocCallOut>

## Kibana role requirements

To create list containers and items, the user role for the ((kib)) space must
have:

* `read` and `write` index privileges for the
    `.lists` and `.items` indices (the system index used for storing exception lists).

* ((kib)) space `All` privileges for the `Security` and `Saved Objects Management`
    features (see
    [Feature access based on user privileges](((kibana-ref))/xpack-spaces.html#spaces-control-user-access)).

See <DocLink id="serverlessSecurityDetectionsPermissionsSection">Detections prerequisites and requirements</DocLink> for a complete list of requirements.
