<div id="windows-user-account-creation"></div>

Identifies attempts to create a Windows User Account. This is sometimes done by attackers to persist or increase access to a system or domain.

**Rule type**: query

**Rule indices**: 

* winlogbeat-*
* logs-system.*
* logs-windows.*

**Severity**: low

**Risk score**: 21

**Runs every**: 5m

**Searches indices from**: None ([Date Math format](((ref))/common-options.html#date-math), see also <DocLink id="serverlessSecurityRulesUiCreate" section="set-the-rules-schedule">`Additional look-back time`</DocLink>)

**Maximum alerts per execution**: 100

**References**: None

**Tags**: 

* Domain: Endpoint
* OS: Windows
* Use Case: Threat Detection
* Tactic: Persistence

**Version**: 1

**Rule authors**: 

* Skoetting

**Rule license**: Elastic License v2

## Rule query

```js
event.module:("system" or "security") and winlog.api:"wineventlog" and
  (event.code:"4720" or event.action:"added-user-account")

```
