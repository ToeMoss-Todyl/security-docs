<div id="multiple-alerts-involving-a-user"></div>

This rule uses alert data to determine when multiple different alerts involving the same user are triggered. Analysts can use this to prioritize triage and response, as these users are more likely to be compromised.

**Rule type**: threshold

**Rule indices**: 

* .alerts-security.*

**Severity**: high

**Risk score**: 73

**Runs every**: 1h

**Searches indices from**: now-24h ([Date Math format](((ref))/common-options.html#date-math), see also <DocLink id="serverlessSecurityRulesUiCreate" section="set-the-rules-schedule">`Additional look-back time`</DocLink>)

**Maximum alerts per execution**: 100

**References**: None

**Tags**: 

* Use Case: Threat Detection
* Rule Type: Higher-Order Rule

**Version**: 3

**Rule authors**: 

* Elastic

**Rule license**: Elastic License v2

## Rule query

```js
signal.rule.name:* and user.name:* and not user.id:("S-1-5-18" or "S-1-5-19" or "S-1-5-20")

```
