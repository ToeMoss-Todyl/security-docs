<div id="multiple-alerts-in-different-att-ck-tactics-on-a-single-host"></div>

This rule uses alert data to determine when multiple alerts in different phases of an attack involving the same host are triggered. Analysts can use this to prioritize triage and response, as these hosts are more likely to be compromised.

**Rule type**: threshold

**Rule indices**: 

* .alerts-security.*

**Severity**: high

**Risk score**: 73

**Runs every**: 1h

**Searches indices from**: now-24h ([Date Math format](((ref))/common-options.html#date-math), see also <DocLink id="serverlessSecurityRulesUiCreate" section="set-the-rules-schedule">`Additional look-back time`</DocLink>)

**Maximum alerts per execution**: 100

**References**: None

**Tags**: 

* Use Case: Threat Detection
* Rule Type: Higher-Order Rule

**Version**: 4

**Rule authors**: 

* Elastic

**Rule license**: Elastic License v2
